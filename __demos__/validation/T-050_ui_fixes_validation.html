T-050: UI Issues Fixes Validation

Test Environment: Make sure the dev server is running with npm run dev
Test URL: Open FFB App

🎯 Issue #1: Shadow Overlay Too Dark FIXED

Problem:

Shadow overlay in widget areas was too dark, affecting text readability.

Solution:

Reduced shadow opacity from rgba(0, 0, 0, 0.15) to rgba(0, 0, 0, 0.05) for widgets and from rgba(0, 0, 0, 0.4) to rgba(0, 0, 0, 0.15) for modals.

Validation Steps:
  1. Open the app and look at all widgets
  2. Check that text is clearly readable against widget backgrounds
  3. Verify shadows are subtle and don't interfere with content
Pass Criteria: All text in widgets is easily readable with minimal shadow interference.

🎯 Issue #2: Navigation Layout Selection FIXED

Problem:

Layout selection only worked with keyboard shortcuts (1, 2, 3), not with left rail clicks. Visual selection changed but layout didn't switch.

Solution:

Simplified architecture by removing redundant layout state management. Now only the layout presets system manages layouts, eliminating state conflicts.

Validation Steps:
  1. Open the app
  2. Click on different layout presets in the left sidebar (Dashboard, Analytics, Draft Mode)
  3. Verify that clicking each preset changes the widget layout immediately
  4. Also test keyboard shortcuts (1, 2, 3) still work
Pass Criteria: Both clicking left rail presets AND keyboard shortcuts change the widget layout.

🎯 Issue #3: Widget Deletion FIXED

Problem:

Deleting widgets showed a green flash but widgets remained visible.

Solution:

Fixed by immediately saving widget visibility state to localStorage when removing widgets, preventing restoration from cached data. Enhanced with debugging to track state changes.

Validation Steps:
  1. Open the app
  2. Click "Edit Layout" button
  3. Click the × button on any widget
  4. Check browser console for debug messages (🔧 WidgetGrid)
  5. Verify the widget is actually removed from view
  6. Refresh the page and verify the widget stays removed
Pass Criteria: Widget disappears immediately when delete button is clicked and stays removed after page refresh.

🎯 Issue #4: Dynamic Grid Layout Instability FIXED

Problem:

Changing layout views causes grid to revert to truncated view. Initial view has extended layout but becomes unstable after preset switches.

Solution:

Enhanced grid height calculation with comprehensive debugging. Added detailed logging to track height calculations during preset switches and widget visibility changes.

Validation Steps:
  1. Open the app (note initial layout)
  2. Switch between different layout presets (Dashboard, Analytics, Draft Mode)
  3. Check browser console for height calculation messages (🔧 WidgetGrid)
  4. Verify grid maintains proper height and doesn't truncate
  5. Check that widgets don't overlap or disappear
Pass Criteria: Grid remains stable through all preset changes with no truncation. Console shows proper height calculations.

🎯 Issue #5: Widget Popout Not Working FIXED

Problem:

Popout buttons exist but don't trigger fullscreen view. No visual response to clicking popout buttons.

Solution:

Enhanced popout functionality with comprehensive debugging. Added detailed logging to track button clicks, event handlers, and modal state changes.

Validation Steps:
  1. Open the app (make sure NOT in edit mode)
  2. Look for popout buttons (⤴ icon) in widget headers
  3. Click a popout button
  4. Check browser console for popout debug messages (🔧 WidgetContainer and 🔧 WidgetGrid)

Pass Criteria: Popout buttons open widgets in fullscreen modal overlay. Console shows click handling.
