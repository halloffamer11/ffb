🧪 Manual Testing Protocol

  1. Load the App: Navigate to http://localhost:5173 (ensure dev server is running)
  2. Screen Reader Test: Enable your OS screen reader (NVDA, JAWS, VoiceOver) and navigate through all widgets
  3. Keyboard Navigation: Use only keyboard to navigate through all interactive elements
  4. Focus Management: Verify visible focus indicators and logical tab order
  5. Performance Test: Monitor React DevTools Profiler during typical usage scenarios
  6. Contrast Check: Verify all text meets WCAG 2.1 AA contrast requirements (4.5:1)
  7. Responsive Test: Test accessibility at different viewport sizes

✅ Validation Criteria

WCAG 2.1 AA Compliance:

  • All interactive elements are keyboard accessible
  • Focus indicators are clearly visible (2px outline)
  • Color contrast meets 4.5:1 minimum ratio
  • All form controls have associated labels
  • Screen readers can navigate and understand all content
  • ARIA labels provide meaningful context
  • No accessibility violations in automated tests

Performance Requirements:

  • Player search responds in under 50ms
  • VBD calculations complete in under 100ms
  • Widget interactions feel smooth (>60fps)
  • No unnecessary component re-renders
  • Memory usage remains stable during use
  • Canvas animations maintain performance
  • Lighthouse Performance score above 90
