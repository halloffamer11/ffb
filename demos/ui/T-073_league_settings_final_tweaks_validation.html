🔧 T-073: League Settings Final Tweaks - HITL Validation

📋 Implementation Summary

Purpose: Final improvements to League Settings page focusing on user team persistence, editable scoring values, draft reset functionality, and compact layout design.

Enhancements Added:

  • User Team Selection Persistence: Fixed user team selection being lost during drag-and-drop team reordering
  • Editable Scoring Values: Made all scoring presets editable with automatic switch to custom when values are modified
  • Reset Draft Button: Added draft reset functionality with undo capability to clear picks while preserving settings
  • Compact Layout: Redesigned page layout with left column (basic settings + roster) and right column (teams) for better space utilization

Integration: All functionality continues to use the unified store with enhanced user experience and better data persistence.

🧪 Validation Instructions

Setup Steps

  1. Start the development server: npm run dev
  2. Open http://localhost:5173
  3. Navigate to League Settings page (gear icon or /league-settings)
  4. Ensure player data is loaded (via Data Management if needed)

Test 1: User Team Selection Persistence During Drag

  1. Select a team (e.g., Team 3) using the radio button
  2. Verify the selected team has visual highlighting
  3. Drag the selected team to a different position in the draft order
  4. Verify the team selection persists after the drag operation
  5. Check that the selected team maintains its visual highlighting

Expected Results:

  • Selected team remains selected even after being dragged to new position
  • Visual highlighting follows the team to its new position
  • Radio button selection state is maintained correctly
  • No loss of user team association during reordering

Test 2: Editable Scoring Values with Auto-Switch to Custom

  1. Set scoring format to "Standard" (receptions should be 0)
  2. Click the expand arrow to show scoring values
  3. Modify the "Receptions" value from 0 to 0.5
  4. Verify that the scoring format automatically switches to "Custom"
  5. Change to "PPR" preset and verify receptions become 1
  6. Modify "Pass TDs" from 4 to 6
  7. Verify automatic switch to Custom preset again
  8. Collapse and expand the scoring section to verify values persist

Expected Results:

  • All scoring values are editable regardless of preset
  • Modifying any value automatically switches preset to "Custom"
  • Preset changes update values, but custom modifications are preserved
  • Scoring section collapses/expands smoothly with current preset shown

Test 3: Draft Reset Button with Undo Functionality

  1. Add some keepers to different teams
  2. Click "Start Draft" to lock settings
  3. Verify all settings become locked and "Reset Draft" button appears
  4. Click "Reset Draft" and confirm the warning dialog
  5. Verify draft is unlocked and keepers/settings remain intact
  6. Check that "Undo Reset" button appears if undo is available
  7. Click "Undo Reset" to restore the draft started state
  8. Verify settings become locked again

Expected Results:

  • Reset Draft button only appears when draft is started
  • Reset clears draft state but preserves league settings and keepers
  • Settings become editable again after reset
  • Undo button appears when undo is available
  • Undo restores exact previous state including draft lock
  • Success messages appear for both reset and undo operations

Test 4: Compact Layout Design

  1. Observe the overall page layout structure
  2. Verify basic settings and roster configuration are in left column
  3. Verify teams management is in the right column
  4. Check that keepers system spans full width below the columns
  5. Resize browser window to test responsiveness
  6. Verify no excessive scrolling is needed to access all sections

Expected Results:

  • Two-column layout with left (settings + roster) and right (teams)
  • Keepers section spans full width below columns
  • More compact design reduces vertical scrolling
  • Layout remains functional on smaller screens
  • Visual balance between sections is maintained

Test 5: Overall Integration and Data Persistence

  1. Make several changes using all the new features
  2. Select a user team, modify scoring, add keepers
  3. Start draft, then reset it, then undo the reset
  4. Refresh the page or navigate away and back
  5. Verify all changes and states persist correctly

Expected Results:

  • All feature interactions work smoothly together
  • Data persistence works correctly across page reloads
  • No conflicts between different functionalities
  • Unified store maintains data integrity

✅ Pass Criteria Checklist

  • User team selection persists through drag operations
  • Scoring values are editable for all presets with auto-switch to custom
  • Scoring section collapses/expands correctly showing current preset
  • Reset Draft button works with proper confirmation dialog
  • Draft reset clears draft data while preserving settings
  • Undo button appears and restores previous state correctly
  • New compact layout improves space utilization
  • Left column contains basic settings + roster configuration
  • Right column contains teams management
  • Keepers system spans full width below columns
  • All data persists correctly across page refreshes
  • No console errors or TypeScript warnings
  • UI remains responsive and accessible
