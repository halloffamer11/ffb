🔧 T-070: Layout Control Buttons Fix Validation

Fixing non-functional layout control buttons in the left sidebar while maintaining keyboard shortcuts

Issue: Edit Layout, Save, and Reset buttons didn't work due to stale closure bugs

🛠 Implementation Summary

Problem: Layout control buttons in left sidebar were using stale closures, capturing old values of editMode when registered.

Root Cause Analysis:

  • Stale Closure: registerLayoutControls was capturing editMode value at registration time
  • Duplicate Systems: Both registerEditModeToggle and registerLayoutControls handling edit mode
  • Navigation Toggle: Same stale closure issue affected navigation toggle

Code Changes:

  • WidgetGrid.tsx: Fixed onToggleEditMode to use stable toggleEditMode callback
  • WidgetGrid.tsx: Fixed temporal dead zone error by moving toggleEditMode definition before usage
  • PresetContext.tsx: Removed duplicate registerEditModeToggle system
  • PresetContext.tsx: Updated 'e' key handler to use layoutControlsRef.current.onToggleEditMode
  • DashboardPage.tsx: Fixed navigation toggle to use functional state update

Files Modified:

  • src/components/widgets/WidgetGrid.tsx - Fixed button registrations
  • src/stores/PresetContext.tsx - Removed duplicate system, fixed keyboard shortcuts
  • src/pages/DashboardPage.tsx - Fixed navigation toggle closure

🔍 Technical Details

Before Fix:

  • Layout control buttons captured stale editMode values
  • Dual registration systems caused confusion
  • Functional state updates not used consistently

After Fix:

  • Single registration system via registerLayoutControls
  • Stable callback functions prevent stale closures
  • Keyboard shortcuts use same handlers as buttons
  • Functional state updates ensure current values

Key Technical Changes:

  • onToggleEditMode: toggleEditMode instead of inline arrow function
  • Removed registerEditModeToggle and editModeToggleRef
  • Keyboard 'e' handler uses layoutControlsRef.current.onToggleEditMode
  • Navigation toggle uses setLeftRailCollapsed(prev => !prev)
