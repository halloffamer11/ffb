Layout Management Features

T-071: Advanced Layout Persistence & Export

HITL VALIDATION REQUIRED

✨ New Features Implemented

  • Preset-Specific Layout Saving
    Manual Save button now saves the current layout to the active preset, persisting across page refreshes and preset switches.
  • Layout Configuration Export
    Clicking Manual Save automatically downloads a JSON configuration file containing the complete layout setup.
  • Enhanced Reset Functionality
    Reset to Default button now properly resets to the original default layout for each preset, clearing all customizations.

📁 Export File Format

Filename: ffb-layout-{presetId}-{date}.json

Contents: Complete layout configuration including widget positions, visibility, and metadata

{ "presetId": "pre-draft", "presetName": "Pre-Draft Research", "layouts": { "lg": [...], "md": [...], "sm": [...] }, "visibleWidgets": ["search", "vbd-scatter", ...], "exportedAt": "2024-01-15T10:30:00.000Z", "version": "1.0.0" }

Purpose: This file can be provided to developers to update default layouts in the codebase.

🧪 Validation Instructions

Setup

  • Dev server should be running on: http://localhost:5174/
  • Clear browser localStorage if needed for clean testing

Test Scenarios

Scenario 1: Preset-Specific Layout Saving
  1. 1A: Start with "Pre-Draft Research" preset (shortcut: 1)
    Expected: Default Pre-Draft layout loads
  2. 1B: Enter Edit Layout mode and customize the layout (move/resize widgets)
    Expected: Layout changes are visible
  3. 1C: Click "Save to Pre-Draft Research" button
    Expected: Button shows current preset name, JSON file downloads automatically
  4. 1D: Refresh the page (F5/Cmd+R)
    Expected: Custom layout persists, not reverted to default
  5. 1E: Switch to "Active Nomination" preset (shortcut: 2)
    Expected: Different default layout for Nomination preset
  6. 1F: Switch back to "Pre-Draft Research" preset (shortcut: 1)
    Expected: Previously saved custom layout is restored
Scenario 2: Layout Export Functionality
  1. 2A: Customize any preset layout and click "Save to [Preset Name]"
    Expected: JSON file automatically downloads to Downloads folder
  2. 2B: Open the downloaded JSON file in a text editor
    Expected: File contains complete layout configuration with proper structure
  3. 2C: Verify file contains: presetId, presetName, layouts (lg/md/sm), visibleWidgets, exportedAt
    Expected: All required fields are present with correct data types
Scenario 3: Enhanced Reset Functionality
  1. 3A: Customize a layout and save it to a preset
    Expected: Layout is saved and persists
  2. 3B: Remove some widgets using the × button in edit mode
    Expected: Some widgets are hidden
  3. 3C: Click "Reset to Default" button
    Expected: Layout reverts to original preset default, all widgets visible
  4. 3D: Refresh the page
    Expected: Default layout persists (custom layout was cleared)

✅ Pass Criteria

  • Each preset maintains its own saved layout independently
  • Manual Save button shows current preset name dynamically
  • JSON configuration file downloads automatically with correct filename format
  • Exported JSON contains all required fields with valid data
  • Saved layouts persist through page refreshes and preset switches
  • Reset to Default properly clears all customizations and restores original layout
  • All widgets become visible after reset (including previously hidden ones)
  • Console shows clear logging of save/load/reset operations
