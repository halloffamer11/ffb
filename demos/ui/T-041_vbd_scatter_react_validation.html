✅ Validation Checklist

1. VBD Calculation Integration

  1. Action: Load the widget and examine the scatter plots
  2. Expected: Each position (QB, RB, WR, TE) shows players with real VBD calculations
  3. Verify: VBD values are not mock data, represent actual baseline calculations
Pass Criteria: VBD values match calculations from src/core/vbd.js, baseline calculations exclude IR players, FLEX baselines applied correctly.

2. Draftable Player Logic

  1. Action: Count players shown per position
  2. Expected: Player counts match draftable thresholds from league settings
  3. Verify: Only top draftable players per position are displayed
Pass Criteria: Player counts per position match expected draftable calculations, uses getDraftablePlayers() logic correctly.

3. Canvas Performance

  1. Action: Toggle position filters rapidly
  2. Expected: Chart updates smoothly without lag or flickering
  3. Verify: Canvas renders 300+ players efficiently
Pass Criteria: All chart updates complete within 100ms, no visual artifacts during rapid position filtering.

4. Interactive Features

  1. Action: Hover over data points
  2. Expected: Tooltips appear with player details (name, position, VBD, team)
  3. Action: Click on data points
  4. Expected: Player selection updates, dots change appearance
Pass Criteria: Tooltips show accurate player information, click selection updates visual state and selectedPlayer store.

5. Cross-Widget Synchronization

  1. Action: Click a player in the scatter plot
  2. Expected: Other widgets reflect the same selected player
  3. Action: Select a player from another widget
  4. Expected: Scatter plot highlights the corresponding dot
Pass Criteria: Player selection synchronizes bidirectionally, emits and responds to player:selected events correctly.

6. Real-Time Draft Updates

  1. Action: Draft a player (simulate pick in draft state)
  2. Expected: Player disappears from scatter plot immediately
  3. Verify: VBD recalculations update remaining players
Pass Criteria: Drafted players removed from chart in real-time, VBD baselines recalculated for remaining undrafted players.

7. Visual Design Compliance

  1. Verify: Position colors match specification (QB: purple, RB: green, WR: blue, TE: orange)
  2. Verify: Professional trading platform aesthetic maintained
  3. Verify: Grid lines, axis labels, and reference lines are clean and readable
Pass Criteria: Visual design matches Robinhood Legend specification, colors consistent across all position filtering states.
