🏈 League Presets Validation - T-055

Purpose: Validate the league preset dropdown functionality including preset loading, smart detection, and custom league behavior. Verify that team names/owners don't trigger "Custom" mode while structural changes do.

📋 Test Setup Instructions

  1. Start the development server: npm run dev
  2. Open the FFB application in your browser
  3. Navigate to League Settings page
  4. Look for the new "🏈 League Template" section at the top
Important: This validation tests preset functionality only. The actual draft functionality is separate and should work with any league configuration.

🧪 Test Cases

Test 1: Basic Preset Loading

  1. Select "CBS Snake" from the League Template dropdown
  2. Observe all settings change automatically
  3. Switch to "Yahoo Auction" preset
  4. Observe settings change to Yahoo configuration
  5. Switch back to "Custom League"
Expected:
• Settings update immediately when preset is selected
• Template info box appears showing which preset is active
• Team count, roster positions, scoring all match preset values
• Flex configuration updates correctly (0 for CBS, 2 for Yahoo)
• Yahoo preset shows 2 flex spots with correct position allowances

Test 2: Team Data Preservation

  1. Start with "Custom League"
  2. Change some team names and owner names
  3. Select "CBS Snake" preset (12 teams)
  4. Verify team names are preserved if possible
  5. Select "Yahoo Auction" preset (10 teams)
  6. Check team names handling with different team count
Expected:
• Custom team names preserved when switching to preset with same team count
• New default teams created when preset has different team count
• Budget values update to match preset
• Preset remains selected (doesn't switch to "Custom")

Test 3: Non-Structural Changes (Should NOT Switch to Custom)

  1. Select "CBS Snake" preset
  2. Change team names and owner names
  3. Select your user team (radio button)
  4. Add/remove keepers (if available)
  5. Change league name
Expected:
• Dropdown remains on "CBS Snake"
• Template info box still shows
• No automatic switch to "Custom League"
• All changes save properly

Test 4: Structural Changes (Should Switch to Custom)

  1. Select "Yahoo Auction" preset
  2. Change team count from 10 to 12
  3. Verify it switches to "Custom League"
  4. Select "CBS Snake" preset again
  5. Change RB positions from 2 to 3
  6. Verify it switches to "Custom League"
  7. Select "Yahoo Auction" preset again
  8. Change receptions scoring from 1.0 to 0.5
Expected:
• Each structural change immediately switches dropdown to "Custom League"
• Template info box disappears
• Settings remain as modified
• User can still select presets to overwrite custom settings

Test 5: Smart Preset Detection

  1. Start with "Custom League"
  2. Manually configure settings to exactly match CBS Snake:
    • 12 teams
    • Standard scoring (0 PPR)
    • QB:1, RB:2, WR:3, TE:1, K:1, DST:1, FLEX:0, BENCH:6
  3. Observe if it auto-detects the preset
  4. Try the same for Yahoo Auction settings
Expected:
• When settings exactly match a preset, dropdown should auto-switch to that preset
• Template info box should appear
• This may happen with a slight delay due to detection logic

Test 6: Flex Position Configuration

  1. Select "Yahoo Auction" preset
  2. Scroll to "🔀 Flex Position Configuration" section
  3. Verify Flex 1 allows: RB ✓, WR ✓, TE ✓, QB ✗
  4. Verify Flex 2 allows: QB ✓, RB ✓, WR ✓, TE ✓
  5. Change Flex 1 to also allow QB
  6. Observe preset behavior
Expected:
• Yahoo preset shows exactly 2 flex spots
• Flex 1 configured as W/R/T (no QB)
• Flex 2 configured as Q/W/R/T (Super Flex)
• Changing flex configuration switches to "Custom League"

Test 7: Draft Lock Behavior

  1. Configure any league settings
  2. Click "Lock Settings" to start draft
  3. Try to change the League Template dropdown
  4. Try to modify other settings
Expected:
• League Template dropdown becomes disabled
• All other settings remain locked as before
• Lock icon appears next to page title
• Template info box remains visible but indicates settings are locked

✅ Validation Checklist

🚨 Common Issues to Check

Potential Problems:
• Preset detection logic may be too sensitive or not sensitive enough
• Flex spot configuration might not update properly
• Team data could be lost unexpectedly
• Smart detection might cause unwanted preset switches
• Performance issues with frequent preset compatibility checks
