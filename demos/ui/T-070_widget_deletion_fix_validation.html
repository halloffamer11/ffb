T-070: Fix Widget Deletion Flash Bug

HITL VALIDATION REQUIRED

🐛 Issue Description

Problem: Widget deletion buttons (×) were nearly impossible to click because the drag handle overlay was covering the entire title bar, including the control buttons.

Root Cause: The drag handle target zone covered the entire title bar, making the × button unclickable since drag events were intercepting click events on the controls area.

🔧 Fix Implementation

Solution: Moved the drag handle from the entire title bar to only the title section, excluding the controls area where the × button is located.

Technical Details:

  • Removed widget-drag-handle class from the entire TitleBar component
  • Applied widget-drag-handle class only to the TitleSection (title + drag dots)
  • Left the Controls section (× button area) completely free from drag handling
  • Added event.stopPropagation() to prevent any remaining event conflicts

🧪 Validation Instructions

Setup

  • Dev server should be running on: http://localhost:5174/
  • Open the FFB Draft Helper application

Test Steps

  1. Step 1: Navigate to the main dashboard page
    Expected: Dashboard loads with multiple widgets visible
  2. Step 2: Click the "Edit Layout" button in the top toolbar
    Expected: Edit mode activates, showing × buttons in widget headers
  3. Step 3: Click the × button ONCE on any widget (e.g., Player Search Widget)
    Expected: Widget disappears immediately with a SINGLE CLICK - no multiple clicking required
  4. Step 4: Wait 2-3 seconds and observe the layout
    Expected: Deleted widget remains gone, other widgets adjust layout smoothly
  5. Step 5: Test dragging widgets by their title area (avoiding the × button)
    Expected: Widgets can still be dragged by clicking on title/drag dots area
  6. Step 6: Delete 2-3 more widgets using their × buttons
    Expected: Each widget deletes with single click, no interference from drag handling
  7. Step 7: Exit edit mode and return to normal view
    Expected: Deleted widgets remain hidden in normal mode
  8. Step 8: Refresh the page completely (F5/Cmd+R)
    Expected: Deleted widgets stay deleted after page reload (localStorage persistence)

✅ Pass Criteria

  • Easy single-click deletion - × button is immediately clickable
  • Widget disappears instantly with no delays or multiple clicks required
  • Drag functionality still works when clicking on title/drag dots area
  • No interference between drag handling and button clicks
  • All widget types can be successfully deleted and dragged independently
  • Deletion state persists through page refreshes

❌ Fail Criteria

  • Any widget flashing (disappear then reappear)
  • Deleted widgets returning after timeouts
  • Console errors during widget deletion
  • Layout breaking or overlapping after deletions
  • Loss of deletion state on page refresh
