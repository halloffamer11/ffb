🧪 Widget Layout Persistence Test

Step-by-step testing for React Grid Layout persistence

📋 Testing Instructions

Step 1: Open the React Application
🚀 Open React App
Step 2: Open Browser Developer Tools
Press F12 or right-click → Inspect Element → Console tab
Step 3: Look for Widget Grid Debug Messages
You should see messages like:
🔧 WidgetGrid: Loading saved layouts... 🔧 WidgetGrid: Saved layouts from storage: null 🔧 WidgetGrid: No saved layouts found, using defaults
Step 4: Enable Edit Mode
Click the "Edit Layout" button in the top-right of the React app
Step 5: Move a Widget
Drag any widget (like Player Search) to a new position by its title bar
Step 6: Check Console for Save Messages
You should see:
🔧 WidgetGrid: Layout changed { _layout: [...], allLayouts: {...} } 🔧 WidgetGrid: Complete layouts to save: {...} 🔧 WidgetGrid: Save result: { ok: true } ✅ WidgetGrid: Layouts saved successfully
Step 7: Test Persistence
Refresh the page (F5) and check if the widget stayed in its new position
Step 8: Check Console After Refresh
After refresh, you should see:
🔧 WidgetGrid: Loading saved layouts... 🔧 WidgetGrid: Saved layouts from storage: { lg: [...], md: [...], sm: [...] } 🔧 WidgetGrid: Merged layouts: {...}

🔍 Debug Information

What to look for if persistence isn't working:

  • Storage not available: Check if localStorage is blocked or full
  • Layout change not triggered: React Grid Layout might not be calling onLayoutChange
  • Save failing: Storage adapter might be encountering quota issues
  • Load failing: Data might be corrupted or in wrong format

Common Issues:

  • Browser in private/incognito mode (localStorage disabled)
  • Site localStorage quota exceeded
  • React Grid Layout not properly configured
  • onLayoutChange only triggers during drag/resize, not on mount

🛠️ Manual Debug Commands

Open browser console and run these commands to debug:

Check localStorage directly:
// Check if layouts are saved Object.keys(localStorage).filter(key => key.includes('widget')) // View the saved layout data JSON.parse(localStorage.getItem('widget-layouts::layouts') || 'null')
Test storage adapter directly:
// Import and test the adapter (run in browser console on the React app page) const { createStorageAdapter } = await import('./src/adapters/storage.js'); const adapter = createStorageAdapter({ namespace: 'widget-layouts', version: '1.0.0' }); console.log('Available:', adapter.isAvailable()); console.log('Current layouts:', adapter.get('layouts'));

Expected Result

After moving widgets and refreshing, they should stay in their new positions.

If this doesn't work: Check the browser console for error messages and debug information.
